# Handshake

## Core Flow

1.  Пользователь создаёт группы.
2.  Пользователь создаёт ноды.
3.  Пользователь добавляет ноды в группы.
4.  Пользователь создаёт контексты.
5.  Пользователь связывает ноды в рамках контекста.
6.  Пользователь создаёт роль, которая имеет доступ к группе.
7.  Пользователь выдаёт роль другому пользователю.

------------------------------------------------------------------------

## Offer

``` yaml
offer:
  id: UUID7

  title: STRING
  expires_at: DATETIME

  to_company: UUID7
  from_company: UUID7

  groups: [UUID7]
  contexts: [UUID7]

  status: PENDING | DECLINED | ACCEPTED

  from_offer: UUID7 | NULL
```

### Ограничения

-   Поле `from_offer` может ссылаться только на исходное предложение.
-   Запрещены цепочки: нельзя создавать `from_offer`, если текущий offer
    уже является ответом (`from_offer != NULL`).

------------------------------------------------------------------------

## Offer Flow (Handshake)

### Шаг 1 --- Компания A

Компания A создаёт предложение (`offer`): - Указывает целевую компанию
(`to_company`) - Указывает группы и контексты, которыми готова
делиться - Статус: `PENDING`

------------------------------------------------------------------------

### Шаг 2 --- Компания B

Компания B: - Принимает предложение A - Создаёт ответное предложение: -
`from_offer` указывает на offer компании A - Указывает свои группы и
контексты

------------------------------------------------------------------------

### Шаг 3 --- Принятие первого предложения

-   Offer компании A переводится в статус `ACCEPTED`
-   Создаётся связь между предложениями (`offer_link`)

------------------------------------------------------------------------

## Offer Link

``` yaml
offer_link:
  id: UUID7

  request_offer: UUID7
  answer_offer: UUID7

  status: ACCEPTED | ABORTED
```

### Семантика

-   `request_offer` --- исходное предложение
-   `answer_offer` --- ответное предложение

------------------------------------------------------------------------

### Шаг 4 --- Финализация

-   Компания A принимает ответное предложение компании B
-   Offer компании B переводится в статус `ACCEPTED`
-   `offer_link` переводится в статус `ACCEPTED`
-   Создаётся контракт
-   У компаний А и B появляются роли позволяющие работать с делимыми данными

------------------------------------------------------------------------

## Contract

``` yaml
contract:
  id: UUID7

  company_a: UUID7
  company_b: UUID7

  company_a_offer: UUID7 | NULL
  company_b_offer: UUID7 | NULL

  status: ACTIVE | EXPIRED | CANCELLED

  expires_in: DATETIME
```

### Ограничения

-   Ровно одно из полей может быть `NULL`
-   Оба поля одновременно не могут быть `NULL`

------------------------------------------------------------------------

## Семантика системы

### Offer

Предложение обмена доступом: - Описывает, какие группы и контексты
сторона готова предоставить - Не раскрывает полную структуру данных

------------------------------------------------------------------------

### Offer Link

Связь между двумя предложениями: - Фиксирует факт двустороннего
согласования

------------------------------------------------------------------------

### Contract

Финализированное соглашение: - Определяет активные отношения между
компаниями - Используется для проверки доступа - Ограничен по времени

------------------------------------------------------------------------

## Общая модель взаимодействия

1.  Одна сторона инициирует доступ (offer)
2.  Вторая сторона отвечает
3.  Формируется offer_link
4.  Создаётся контракт
5.  Контракт используется для контроля доступа
